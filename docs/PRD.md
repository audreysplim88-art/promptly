# Promptly — Product Requirements Document

**Version:** 1.0  
**Date:** April 2026  
**Status:** Phase 1 shipped

---

## 1. Problem

AI tools like ChatGPT, Claude, and Gemini are only as good as the prompts they receive. The vast majority of users — from home cooks to software engineers — write prompts that are too vague, lack context, and omit the constraints and format instructions that would dramatically improve the response.

The result: users get mediocre outputs, blame the AI, and miss the real value these tools can deliver.

**The gap:** Prompt engineering is a skill. Most users don't have it and don't want to learn it. They just want a great answer.

---

## 2. Solution

Promptly is a browser extension that acts as a prompt engineer on the user's behalf. It intercepts the moment before someone types a prompt and replaces guesswork with a structured, expert-guided interview.

**Core flow:**
1. User clicks the Promptly button on any supported AI tool page
2. States their goal in plain language
3. Answers 4–6 targeted questions generated by Claude
4. Receives a polished, expert-quality prompt
5. Inserts it directly into the AI tool's text box with one click

## 5. Features

### Phase 1 — Core (shipped)

- **In-page overlay panel** slides in from the right on supported AI tool pages
- **Goal input** — plain-language text box, no structure required
- **Output style selector** — three modes chosen before submitting a goal:
  - *Standard* — full 10-technique prompt engineering, 100–250 words
  - *Concise* — essential techniques only (role, context, format, constraints), 60–120 words
  - *Developer* — imperative task/stack/constraints/format/verification structure for AI coding tools, 60–120 words
- **Domain detection** — Claude classifies the goal as general / creative / technical / professional with confidence score; user can override
- **Interview engine** — Claude generates 4–6 targeted questions (text, radio, checkbox) specific to the user's goal
- **Prompt synthesis** — Claude applies prompt engineering techniques appropriate to the selected style, streamed progressively
- **One-click injection** — inserts the generated prompt directly into the AI tool's textarea
- **Copy to clipboard** — fallback for unsupported tools
- **API security** — shared secret header (`X-Promptly-Secret`) protects backend routes
- **Supported tools** — ChatGPT, Claude, Gemini, Perplexity, Microsoft Copilot

### Phase 2 — Auth + distribution

- User accounts via Clerk (OAuth)
- Free tier: 10 generated prompts per day
- Usage meter displayed in the panel
- Chrome Web Store public listing
- Firefox AMO release

### Phase 3 — Monetisation

- Pro tier ($8/month) — unlimited prompts
- Stripe Checkout + subscription management
- Upgrade CTA shown when free limit is reached
- Stripe webhook updates user tier in Supabase

### Phase 4 — Polish

- Prompt history — browse and re-use past generated prompts
- Keyboard shortcut to open/close the panel (⌘⇧P)
- Prompt rating system — thumbs up/down feeds quality improvement loop
- Domain-specific question templates for common use cases

## 6. Prompt engineering approach

The core IP is two Claude system prompts:

**Interview system prompt** instructs Claude to:
- Output structured JSON (domain, confidence, questions array)
- Generate goal-specific questions — never generic ones
- Cover 4–6 prompt engineering dimensions: goal, context, audience, format, constraints, tone
- Adapt question count and type by domain (technical goals get more questions)

**Synthesis system prompt** selects from three output styles at generation time:

*Standard* — applies all 10 techniques:
1. Role assignment — specific expert persona
2. Context setting — all background before the task
3. Audience specification — who the output is for
4. Task decomposition — numbered steps for complex goals
5. Output format specification — explicit markdown/bullets/code/table
6. Constraints and boundaries — what NOT to do
7. Tone and style — especially for creative/professional
8. Examples instruction — "show, don't tell"
9. Verification step — for technical: review for edge cases
10. Closing kickoff — end with "Begin by..." to prevent meta-commentary

*Concise* — applies role, context, format, and constraints only; hard word ceiling of 60–120 words.

*Developer* — no role preamble; imperative language; structured as Task → Stack → Constraints → Output Format → Verification; 60–120 word ceiling.

## 7. Technical architecture

```
[User on AI tool page]
        ↓ clicks Promptly button
[Plasmo CSUI — overlay.tsx]
        ↓ POST /api/interview
[Next.js on Vercel]
        ↓ Claude API
[Interview questions returned as JSON]
        ↓ user answers questions
[Plasmo CSUI]
        ↓ POST /api/synthesize (streaming SSE)
[Next.js on Vercel]
        ↓ Claude API (streaming)
[Polished prompt streamed to panel]
        ↓ user clicks "Insert into chat"
[Content script injects text into AI tool textarea]
```

**Extension:** Plasmo (React, TypeScript, Manifest V3)  
**Backend:** Next.js 16, deployed on Vercel  
**AI:** Anthropic Claude API (`claude-sonnet-4-6`)  
**Auth (Phase 2):** Clerk  
**Database (Phase 2):** Supabase (Postgres)  
**Payments (Phase 3):** Stripe  

---

## 8. Non-goals

- **Not a standalone AI chatbot** — Promptly generates prompts for other AI tools; it doesn't answer questions itself
- **Not a prompt library** — prompts are generated fresh per goal, not selected from templates
- **Not a writing assistant** — Promptly improves the question, not the answer
- **No model switching** — users use their own AI tool of choice; Promptly is tool-agnostic

---

## 9. Constraints and risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| AI tool UI changes break textarea injection | Medium | Per-tool selectors in `constants.ts`; monitor and update on breaking changes |
| Chrome Web Store review rejection | Low | Permissions are minimal and justified; no remote code execution |
| Claude API latency feels slow | Medium | Stream the synthesis response progressively; interview questions are fast |
| Users don't understand the value | Medium | Onboarding copy on the goal input step; example suggestions |
| API costs at scale | Medium | Free tier limit (10/day) caps cost; Pro tier revenue offsets |
